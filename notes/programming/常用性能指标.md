# 常用指标

- 响应时间 RT（Response-Time）：从请求发出到收到响应的时间差。
- **TPS**
    - Transactions Per Second
    - 使用 TPS 概念时要先定义 T 的含义，可定义为接口级事务、业务级事务等
    - TPS 反应 T 定义层面的处理能力，若定义为接口级，反应这个接口的处理能力
- QPS
    - 原本用于描述 MySQL 的每秒 SQL 执行数（所有的 SQL 都称为 Query）
    - 两种含义
        - 数据库每秒执行 SQL 数
        - 每秒查询数，不包括插入、更新、删除（可以描述前端层或接口层）
    - 不适合描述系统的整体处理能力
- 活跃度
    - PV：页面访问量
    - UV：独立用户访问量
    - DAU：日活跃用户
    - MAU：月活跃用户

# 区分 TPS、线程数、用户数、并发量概念

并发：系统同时能处理多少个请求，每秒系统能同时处理多少请求。常用 TPS 来描述。

有 10000 在线用户，不代表每秒有 10000 的 TPS，在线用户并不是都正在使用同一业务，而且在线并不代表正在操作，并发比例低于 5%：`10000 * 5% = 500(TPS)`。

500 TPS 也不代表要在压测工具中设置 500 个线程同时请求，还要考虑每个线程每秒完成事务数，若每个线程一秒能完成 10 个事务（可通过 RT 计算出，假设响应时间为 100ms，1000ms/100ms = 10 个事务），只需设置 500/10 = 50 个线程即可。

总结：

- 在线用户数不等于并发量
- 并发量不等于线程数
- 并发量即每秒能同时处理多少请求，常用 TPS 描述并发量
