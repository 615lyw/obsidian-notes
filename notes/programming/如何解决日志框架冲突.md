# 为什么会出现日志框架冲突

本质是因为项目中引用多个日志框架。

- Maven 的传递性依赖（**最常见**）
- 手动显示在 POM 中引用多个日志框架
- 引用同一日志框架不同版本

# 解决步骤

1. 找出项目中引入的所有日志框架
2. 根据公司日志规范决定项目中最终使用的日志门面+日志实现框架
3. 排除其他日志实现框架的传递性依赖
4. 通过下图进行日志框架的适配/桥接

# 日志适配解决大全

上层 3 个是日志门面，下层 5 个是日志实现。

日志门面 -> 日志实现：日志门面使用此日志实现（适配）
日志实现 -> 日志门面：把日志实现 API 转换为日志门面 API（桥接）
